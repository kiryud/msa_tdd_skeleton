# Interview Question 01

## 주제

### Garbage Collection(GC)이란 무엇인가?

---

## 질문

Garbage Collection 이란 무엇일까요?

Garbage Collection 방식 중 하나를 설명해주세요.

---

## 답변

Garbage Collection(GC)은 프로그램이 동적으로 할당한 메모리 영역 중에서 더 이상 사용되지 않는 객체를 자동으로 탐지하여 메모리를 회수하는 기능이다.

개발자가 직접 메모리를 해제하지 않아도 되므로 생산성과 안정성을 높일 수 있으며, Java Virtual Machine(JVM)이 메모리 관리를 담당한다.

---

## G1 GC (Garbage First Garbage Collector)

현재 Java 9 이상에서 기본 GC로 사용되고 있으며, Java 17 및 Spring Boot 3.x 환경에서도 널리 사용된다.

### 주요 특징

#### 1. 영역 기반 힙 구조

기존 Young/Old 영역 중심 구조와 달리 Heap을 여러 개의 Region으로 분할하여 관리한다.

```text
Heap

├── Region A
├── Region B
├── Region C
├── Region D
└── ...
```

필요한 Region만 선택적으로 정리할 수 있다.

---

#### 2. 병렬 및 동시 수행

GC 작업을 여러 CPU 코어에서 병렬로 수행한다.

또한 일부 작업은 애플리케이션 실행과 동시에 수행하여 Stop-The-World 시간을 줄인다.

---

#### 3. 우선순위 기반 메모리 회수

가비지가 많이 존재하는 Region을 우선적으로 정리한다.

이를 통해 전체 성능을 향상시킨다.

---

#### 4. 낮은 응답 지연(Low Pause Time)

대규모 Heap 환경에서도 비교적 짧은 GC 정지 시간을 유지할 수 있다.

---

## Java 17 설정 예시

```bash
-XX:+UseG1GC
```

# 심화 질문

## Garbage Collection 언어를 원자력 발전소나 국가 전력망 제어 시스템에 사용할 수 있을까?

---

## 답변

사용은 가능하지만 매우 신중하게 접근해야 한다고 생각한다.

---

### 이유 1. 실시간 응답 보장 문제

원자력 발전소, 자동차 제어 시스템, 인공위성 제어 시스템 등은 실시간(Real-Time) 응답이 매우 중요하다.

그러나 Garbage Collection은 실행 시점이 완전히 예측 가능하지 않으며, GC 수행 중 짧은 시간이라도 애플리케이션이 중단될 수 있다.

이러한 지연은 일반 웹 서비스에서는 큰 문제가 아닐 수 있지만, 실시간 제어 시스템에서는 치명적인 문제로 이어질 수 있다.

---

### 이유 2. 메모리 제어의 한계

GC 기반 언어는 메모리 관리를 JVM이 수행한다.

따라서 개발자가 메모리 사용량을 매우 세밀하게 제어하기 어렵다.

실시간 시스템은 메모리 사용량 자체가 중요한 경우가 많다.

---

### 이유 3. 성능 오버헤드

GC 자체도 CPU 자원을 사용한다.

시스템 전체 자원이 매우 제한된 환경에서는 성능 저하 요인이 될 수 있다.

---

## 결론

GC 기반 언어는 실시간 시스템에 반드시 사용할 수 없는 것은 아니다.

다만 다음 조건이 충족되어야 한다.

### 필요 조건

- 적절한 GC 선택
- GC 튜닝
- 실시간 요구사항 분석
- 충분한 성능 검증

실제로 최근에는 Java Real-Time 기술과 고성능 GC 기술의 발전으로 일부 실시간 시스템에서도 Java를 사용하는 사례가 존재한다.

# 프로젝트 연계

## User Service와의 관계

현재 프로젝트의 User Service는 Spring Boot + Kotlin 기반으로 개발된다.

Spring Boot는 JVM 위에서 동작하므로 Garbage Collection의 영향을 받는다.

---

### 실제 예시

회원가입 요청

↓

User 객체 생성

↓

DTO 생성

↓

GraphQL Response 생성

↓

GC 대상 객체 발생

---

즉 User Service에서 생성되는 객체들은 JVM Heap에 저장되며, 사용이 끝난 객체는 Garbage Collection에 의해 자동으로 회수된다.

---

# 발표 예상 질문

## Q1. Garbage Collection이 필요한 이유는 무엇인가요?

### 답변

개발자가 직접 메모리를 해제하지 않아도 되므로 메모리 누수 위험을 줄이고 생산성을 높일 수 있기 때문입니다.

---

## Q2. Spring Boot는 어떤 GC를 사용하나요?

### 답변

Java 9 이상에서는 기본적으로 G1 GC가 사용됩니다.

---

## Q3. G1 GC의 장점은 무엇인가요?

### 답변

Heap을 Region 단위로 관리하여 효율적으로 메모리를 회수하고, 짧은 응답 지연 시간을 제공한다는 장점이 있습니다.

---

## Q4. User Service에서도 GC가 동작하나요?

### 답변

네. 회원가입 DTO, 로그인 요청 객체, GraphQL 응답 객체 등은 Heap 영역에 생성되며 사용이 끝나면 GC에 의해 회수됩니다.

---

# 실무 관점

실제 Spring Boot 백엔드 서비스에서는 GC 튜닝이 매우 중요하다.

특히 대규모 트래픽 환경에서는 Heap 크기와 GC 설정에 따라 응답 속도와 서버 안정성이 크게 달라질 수 있다.

---

# 한 줄 요약

Garbage Collection은 JVM이 사용하지 않는 객체를 자동으로 회수하는 메모리 관리 기술이며, Java 17과 Spring Boot 환경에서는 G1 GC가 기본적으로 사용되어 효율적인 메모리 관리를 수행한다.
