## User Service 코드 설명

### UserRegisterDto.kt

회원가입 시 전달되는 사용자 정보를 하나의 객체로 묶기 위해 작성.

실생활로 비유하면 회원가입 신청서 같은 역할.

사용자가 입력한 아이디, 비밀번호, 닉네임, 동네 정보를 한 번에 담아서 전달하는 목적.

Kotlin Data Class를 사용하여 불필요한 보일러플레이트 코드를 줄이고 객체를 간결하게 관리.

---

### UserService.kt

로그인 요청을 처리하고 세션 토큰을 발급하는 비즈니스 로직 역할.

실생활로 비유하면 아파트 경비실이나 출입 게이트와 비슷하다고 생각.

입주민인지 확인한 뒤 출입증을 발급해 주는 것처럼 로그인 정보를 검증하고 토큰을 발급하는 구조.

Coroutine을 적용하여 타임딜 마감 직전 대량의 로그인 요청이 발생하더라도 스레드 점유를 최소화하는 명분 설계.

현재는 의사코드 수준으로 작성하였으며 추후 MySQL 계정 검증 및 Redis 세션 저장 로직 추가 가능.

---

### UserQueryResolver.kt

GraphQL 환경에서 사용자 프로필 조회 요청을 처리하기 위한 Resolver 역할.

실생활로 비유하면 주민센터 민원 창구와 비슷하다고 생각.

사용자가 필요한 정보만 요청하면 필요한 정보만 조회해서 전달.

GraphQL을 사용하는 이유는 화면마다 필요한 데이터가 다르기 때문.

메인 화면에서는 닉네임만 필요할 수 있고, 프로필 화면에서는 추가 정보가 필요할 수 있음.

REST API의 오버패칭 문제를 줄이기 위한 명분 설계.

---

### UserServiceTest.kt

User Service 기능이 정상 동작하는지 확인하기 위한 Small Test 코드.

실생활로 비유하면 자동차 출고 전 품질 검사 과정과 비슷.

실제 사용자에게 기능을 제공하기 전에 정상 동작 여부를 미리 확인하는 목적.

현재 검증 내용

- 로그인 성공 시 토큰 발급 확인
- 빈 아이디 입력 시 예외 발생 확인
- 빈 비밀번호 입력 시 예외 발생 확인

TDD 문서에서 정의한 테스트 시나리오를 코드로 옮긴 예시.
