# User Service Large Test

Large Test는 실제 운영 환경과 유사한 End-to-End 시나리오로, 로그인부터 세션 발급·프로필 조회까지 전체 흐름을 검증한다.

## 테스트 1. 사용자 로그인 시나리오

```text
// Given - 등록된 사용자 정보와 올바른 비밀번호가 있다.
// When  - 로그인 요청을 수행한다.
// Then  - 사용자 인증에 성공하고 세션 토큰이 발급되어 로그인 상태로 진입해야 한다.
```

## 테스트 2. Redis 세션 저장 시나리오

```text
// Given - 로그인에 성공한 사용자가 있다.
// When  - 세션 토큰이 발급된다.
// Then  - Redis에 세션 정보가 저장되어 인증 상태가 유지되어야 한다.
```

## 테스트 3. GraphQL 프로필 조회 시나리오

```text
// Given - 로그인을 완료하고 유효한 세션 토큰을 보유한 사용자가 있다.
// When  - GraphQL Query로 프로필을 요청한다.
// Then  - nickname, baseLocation이 담긴 UserRegisterDto가 반환되어야 한다.
// Then  - 비밀번호 정보는 반환되지 않아야 한다.
```

## 테스트 4. 인증 실패 시나리오

```text
// Given - 등록된 사용자에 대해 잘못된 비밀번호가 입력된다.
// When  - 로그인 요청을 수행한다.
// Then  - 인증에 실패하고 토큰이 발급되지 않으며 세션도 생성되지 않아야 한다.
```

## 테스트 5. 세션 만료 시나리오

```text
// Given - 로그인을 완료하고 Redis에 세션이 저장되어 있다.
// When  - 세션 TTL이 만료된다.
// Then  - Redis 세션이 삭제되고 재로그인이 필요해야 한다.
```

## 테스트 6. 전체 사용자 흐름 검증

```text
// Given - 플랫폼 사용자가 있다.
// When  - 로그인 -> 세션 토큰 발급 -> Redis 세션 저장 -> GraphQL 프로필 조회 순으로 진행한다.
// Then  - 전체 과정이 정상 수행되고 인증 상태가 유지되며 프로필 조회가 가능해야 한다.
```
