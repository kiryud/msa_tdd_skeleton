# 타임딜 공동구매 Large Test

Large Test는 여러 기능과 저장소, 알림 요청이 연결된 전체 흐름을 검증하는 테스트입니다.

## 테스트 1. 동시에 여러 명이 참여해도 최대 인원을 초과하지 않는다

```text
// Given - 최대 참여 인원이 50명인 공동구매가 열려 있다.
// When - 100명의 사용자가 동시에 참여 요청을 보낸다.
// Then - Redis에 저장된 참여자는 최대 50명을 넘지 않아야 한다.
// Then - 50명 이후의 요청은 DEAL_FULL 결과를 반환해야 한다.
```

## 테스트 2. 동시에 같은 사용자가 여러 번 참여해도 한 번만 등록된다

```text
// Given - 하나의 userId가 같은 공동구매에 여러 번 참여 요청을 보낸다.
// When - 요청들이 거의 동시에 처리된다.
// Then - Redis 참여자 목록에는 해당 userId가 한 번만 저장되어야 한다.
// Then - 나머지 요청은 ALREADY_JOINED 결과를 반환해야 한다.
```

## 테스트 3. 마감 시간이 지나면 공동구매 성공 상태로 확정된다

```text
// Given - 최소 참여 인원이 10명이고 현재 참여 인원이 12명이다.
// Given - 현재 시간이 공동구매 마감 시간 이후이다.
// When - closeTimeDeal 함수가 실행된다.
// Then - 공동구매 상태는 SUCCESS로 변경되어야 한다.
// Then - 참여자에게 성공 알림 요청이 전달되어야 한다.
```

## 테스트 4. 최소 인원이 부족하면 공동구매 실패 상태로 확정된다

```text
// Given - 최소 참여 인원이 10명이고 현재 참여 인원이 7명이다.
// Given - 현재 시간이 공동구매 마감 시간 이후이다.
// When - closeTimeDeal 함수가 실행된다.
// Then - 공동구매 상태는 FAILED로 변경되어야 한다.
// Then - 참여자에게 실패 알림 요청이 전달되어야 한다.
```

## 테스트 5. 참여부터 마감 확정까지 전체 흐름이 연결된다

```text
// Given - 공동구매가 OPEN 상태로 생성되어 있다.
// When - 사용자들이 참여 요청을 보내고 최소 인원을 달성한다.
// When - 마감 시간이 지난 뒤 closeTimeDeal 함수가 실행된다.
// Then - Redis 참여자 수와 RDBMS 참여 이력이 일치해야 한다.
// Then - 공동구매 상태는 SUCCESS로 확정되어야 한다.
```
