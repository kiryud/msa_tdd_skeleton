# 06주차 정리

## 학습 주제

### AI 시대, 나는 어떤 엔지니어가 될 것인가?

AI(ChatGPT, GitHub Copilot 등)의 발전으로 인해 단순 코드 작성 능력만으로는 개발자의 경쟁력을 설명하기 어려워졌다.

이번 주차에서는 AI와 개발자의 역할을 비교하고, 앞으로 개발자가 갖추어야 할 핵심 역량에 대해 토론하였다.

---

# 핵심 질문

## 1. AI를 사용해보며 가장 편리했던 점은 무엇인가?

### 장점

- 코드 자동 생성
- 반복 작업 감소
- 빠른 정보 검색
- 학습 속도 향상
- 예제 코드 제공

---

### 실제 경험

ChatGPT를 활용하여

- Kotlin 문법 학습
- GraphQL 구조 이해
- Spring Boot 설계 검토
- README 문서 작성

등의 작업 시간을 크게 단축할 수 있었다.

---

# 2. AI의 한계는 무엇인가?

### 문제점

- 존재하지 않는 API 생성
- 오래된 정보 제공
- 프로젝트 문맥 이해 부족
- 잘못된 아키텍처 제안
- 비즈니스 요구사항 오해

---

### 실제 사례

AI가 생성한 코드는 컴파일은 되지만,

프로젝트의 실제 요구사항이나 비즈니스 로직과 맞지 않는 경우가 존재한다.

---

## 결론

AI는 매우 강력한 도구이지만 최종 설계와 판단은 개발자의 책임이다.

---

# AI보다 개발자가 더 잘할 수 있는 것

## 1. 비즈니스 이해

기술보다 중요한 것은 서비스가 해결하려는 문제를 이해하는 능력이다.

---

### 예시

우리동네 기반 타임딜 공동구매 플랫폼

AI는 회원가입 기능을 구현할 수 있다.

그러나

```text
왜 공동구매 서비스가 필요한가?

왜 위치 인증이 필요한가?

왜 Redis를 사용하는가?
```

와 같은 비즈니스 의사결정은 개발자가 수행해야 한다.

---

## 2. 협업 능력

실제 프로젝트는 혼자 개발하는 것이 아니다.

---

### 필요한 역량

- 의사소통
- 일정 조율
- 코드 리뷰
- 역할 분담

---

### 프로젝트 적용

본 프로젝트에서는

- Jira
- GitHub
- README
- 발표 준비

등을 통해 협업 경험을 수행하였다.

---

## 3. 책임감

AI가 생성한 코드의 책임은 AI가 아닌 개발자에게 있다.

---

### 예시

회원 비밀번호 저장

↓

AI가 평문 저장 코드 생성

↓

보안 사고 발생

↓

책임은 개발자에게 있음

---

따라서 결과를 검증하고 책임지는 능력이 중요하다.

---

## 4. 창의적 문제 해결

AI는 기존 데이터를 기반으로 답변한다.

반면 개발자는 새로운 문제를 정의하고 해결책을 설계할 수 있다.

---

### 예시

우리동네 기반 타임딜 공동구매 플랫폼

↓

하이퍼로컬 공동구매

↓

위치 인증

↓

Redis 기반 대기열

↓

GraphQL 최적화

---

이러한 서비스 설계는 개발자의 창의적 사고에서 시작된다.

# 발표 예상 질문

## Q1. AI가 코드를 잘 짜는데 개발자는 왜 필요한가요?

### 답변

AI는 코드를 생성할 수 있지만 서비스 요구사항 분석, 비즈니스 이해, 책임 있는 의사결정은 수행할 수 없기 때문입니다.

---

## Q2. AI보다 개발자가 더 중요한 역량은 무엇이라고 생각하나요?

### 답변

비즈니스 이해 능력과 문제 해결 능력이라고 생각합니다.

기술은 AI가 보조할 수 있지만 어떤 문제를 해결할 것인지는 사람이 결정해야 합니다.

---

## Q3. 이번 프로젝트에서 AI를 어떻게 활용했나요?

### 답변

Kotlin, Spring Boot, GraphQL 학습과 README 작성에 활용하였으며, User Service 설계 검토에도 참고 자료로 활용하였습니다.

---

## Q4. AI가 작성한 코드를 그대로 사용해도 되나요?

### 답변

아닙니다.

AI는 잘못된 코드나 오래된 정보를 제공할 수 있으므로 반드시 개발자가 검증해야 합니다.

---

# 실무 관점

AI 시대의 개발자는 단순 구현자가 아니라 문제 해결자(Problem Solver) 역할을 수행해야 한다.

앞으로는 코드 작성 능력보다

- 설계 능력
- 비즈니스 이해
- 협업 능력
- 시스템 사고

가 더욱 중요해질 것으로 예상된다.
