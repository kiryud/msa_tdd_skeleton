# 핵심 개념

## Git

Git은 분산 버전 관리 시스템(Distributed Version Control System)으로 소스코드 변경 이력을 관리하고 협업을 가능하게 하는 도구이다.

### 주요 기능

- Commit
- Push
- Pull
- Branch
- Merge

### Git 사용 목적

- 코드 백업
- 버전 관리
- 협업 환경 구축
- 충돌 관리

---

## GitHub

Git 저장소를 원격으로 관리하는 플랫폼이다.

### GitHub 사용 목적

- 소스코드 공유
- 협업
- 코드 리뷰
- 프로젝트 기록 관리

---

## Jira

프로젝트를 이슈 단위로 관리하는 협업 도구이다.

### Jira 주요 개념

#### Epic

큰 단위의 프로젝트 목표

예시

```text
User Service
TimeDeal Service
Catalog Service
```

---

#### Story

사용자 관점의 기능 요구사항

예시

```text
회원가입 기능
로그인 기능
```

---

#### Task

실제 개발 작업 단위

예시

```text
User Entity 설계
JWT 인증 구현
```

---

#### Bug

개발 과정에서 발생한 오류 수정 작업

# 프로젝트 적용

## 프로젝트 주제

### 우리동네 기반 타임딜 공동구매 플랫폼

사용자의 현재 위치(GPS)를 기반으로 같은 지역 주민들끼리 공동구매를 진행하거나 배달비를 절약할 수 있도록 지원하는 하이퍼로컬(Hyperlocal) 플랫폼이다.

---

## Git 적용

프로젝트는 팀 단위 협업으로 진행되므로 각 팀원은 독립적인 Feature Branch를 생성하여 개발을 진행한다.

예시

```text
feature/daoul
feature/user-service
```

이를 통해 다른 팀원의 작업과 충돌을 최소화할 수 있다.

---

## GitHub 적용

모든 소스코드를 중앙 저장소에서 관리한다.

### 활용 목적

- 협업
- 코드 공유
- 변경 이력 추적
- 백업

---

## Jira 적용

프로젝트 업무를 Jira 이슈로 관리한다.

예시

```text
KAN-24 User Service
KAN-21 AWS Architecture Diagram
KAN-30 Container Environment
```

담당자를 지정하여 업무 책임 범위를 명확하게 구분한다.

# 발표 예상 질문

## Q1. Git을 사용한 이유는 무엇인가요?

### 답변

여러 명의 팀원이 동시에 개발하는 환경에서 소스코드 버전 관리와 협업을 효율적으로 수행하기 위해 사용하였습니다.

---

## Q2. GitHub를 사용한 이유는 무엇인가요?

### 답변

원격 저장소를 통해 소스코드를 공유하고 변경 이력을 관리하며 협업 효율을 높이기 위해 사용하였습니다.

---

## Q3. Jira를 사용한 이유는 무엇인가요?

### 답변

프로젝트 업무를 이슈 단위로 관리하고 담당자를 지정하여 협업 효율과 업무 추적성을 향상시키기 위해 사용하였습니다.

---

## Q4. User Service 작업은 어떻게 관리했나요?

### 답변

Jira를 통해 User Service 관련 업무를 관리하였으며, Git Feature Branch 전략을 적용하여 독립적으로 개발을 진행하였습니다.
